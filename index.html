API для чёрного экрана

Методы `window.androidApi`, входящие события `window.onAndroidEvent`, команды `runEnum` и JSON-ответы для HTML-проектов чёрного экрана.

Чёрный экран Приборная панель

Подключение

В WebView доступен объект `window.androidApi`. Почти все вызовы принимают первым параметром токен.

Токен

const TOKEN = "SECURE_TOKEN_2025";

Передаётся первым аргументом в методы Android API. При неверном токене методы возвращают ошибку, пустой ответ или ничего не делают.

Готовность страницы

window.androidApi.onJsReady(TOKEN)

Страница вызывает этот метод после инициализации. После этого Android начинает отправлять события в `window.onAndroidEvent`.

JS → Android

Методы вызываются из HTML/JS через `window.androidApi`.

Закрытие и настройки

onClose(token)

Сообщает Android, что пользователь нажал кнопку закрытия чёрного экрана.

onSettings(token)

Открывает настройки текущего веб-экрана.

Команды приложения

runEnum(token, enumName)

Запускает встроенную команду по имени из `AppEnums.CarRunApp`. Возвращает JSON со статусом или ошибкой.

getRunEnum(token)

Возвращает JSON-список доступных enum-команд.

getRunEnumPic(token, enumName)

Возвращает картинку команды в base64.

Запуск приложений

getUserApps(token)

Возвращает массив пользовательских приложений: `name`, `package`, `icon` в base64.

runApp(token, packageName)

Запускает внешнее приложение по package name.

getAppInfo(token, packageName)

Возвращает название и иконку приложения.

Громкость

setvol(token, value)

Устанавливает громкость через `UtilsAll.setVol(...)`.

getvol(token)

Возвращает текущую media-громкость в процентах `0..100`; при ошибке возвращает `-1`.

Данные авто

getCarData(token, name)

Единая точка получения JSON-данных автомобиля.

Доступные имена запроса: `frontSeats`, `front_seats`.

передние сиденья

Файлы и логи авто

getFileList(token)

Возвращает JSON-список файлов из Downloads.

getFile(token, name)

Возвращает файл в base64.

setLogFilter(token, q1, q2, q3)

Задаёт до трёх фильтров для чтения строк из автомобильных логов. Подходящие строки приходят событием `LogData`.

Служебные методы

pingEvent(token)

Отправляет обратно JS-событие `ping` с `{ "ok": true }`.

setBright(token, value)

Принимает значение `0..100` для яркости/прозрачности экрана.

runJson(token, json)

Резервный метод для сложных JSON-команд. Возвращает JSON-статус.

служебные методы

Получение данных

Контракты методов, которые возвращают данные в JS. Если метод возвращает строку, её нужно разобрать через `JSON.parse(...)`.

Громкость

getvol(token) → number

Возвращает уровень media-громкости в процентах. Ошибка: `-1`.

85

Передние сиденья

getCarData(token, "frontSeats") → JSON string

`seat: 1` — левое/водительское, `seat: 2` — правое/пассажирское. `heat` и `vent` — текущие уровни.

{
  "name": "frontSeats",
  "frontLeft": {
    "seat": 1,
    "heat": 2,
    "vent": 0
  },
  "frontRight": {
    "seat": 2,
    "heat": 0,
    "vent": 1
  }
}

Список приложений

getUserApps(token) → JSON string

Массив приложений, доступных для запуска из слотов.

[
  {
    "name": "Навигатор",
    "package": "ru.yandex.yandexnavi",
    "icon": "iVBORw0KGgo..."
  },
  {
    "name": "Музыка",
    "package": "com.example.music",
    "icon": "iVBORw0KGgo..."
  }
]

Информация о приложении

getAppInfo(token, packageName) → JSON string

Название и иконка одного приложения.

{
  "name": "Навигатор",
  "icon": "iVBORw0KGgo..."
}

Список команд

getRunEnum(token) → JSON string

Список команд `runEnum` с отображаемыми именами и служебными данными.

[
  {
    "name": "MEDIA_PLAY",
    "displayName": "Медиа плей",
    "icon": 2131230000,
    "run": 14
  }
]

Иконка команды

getRunEnumPic(token, "MEDIA_PLAY") → string

Base64-строка картинки. Её можно подставить в `src` через `data:image/png;base64,...`.

iVBORw0KGgoAAAANSUhEUg...

Список файлов

getFileList(token) → JSON string

Файлы из Downloads, доступные для чтения через API.

[
  {
    "name": "theme.zip",
    "size": 88340
  }
]

Файл

getFile(token, name) → string

Base64-содержимое файла. Если файл не найден или токен неверный, возвращается пустая строка.

UEsDBBQAAAAIA...

Ошибки JSON-методов

Методы, возвращающие JSON-строку, при ошибке обычно возвращают объект с полем `error`.

{
  "error": "unauthorized"
}

Android → JS

Android вызывает `window.onAndroidEvent(type, data)`. Значение `type` определяет формат объекта `data`.

Событие JSON data Назначение
musicInfo PlayStat, SongName, SongArtist, SongAlbum, SongAlbumPicture, Trpos, Trdur Обновление плеера: название, артист, обложка, прогресс и длительность трека.
gps lat, lon, speed, heading Координаты, скорость GPS и курс.
speed value Скорость автомобиля.
rpm value Обороты двигателя.
hud hudSenderType, aradarOn, naviOn, turnType, turnDist, remainDist, nextRoad, speedLimit Навигационные и антирадарные данные для HUD/виджетов.
weather temp, icon Температура и ID иконки погоды.
theme mode: dark или light Тема WebView после `onJsReady`.
LogData value Строка из автомобильного лога, найденная по фильтрам `setLogFilter(token, q1, q2, q3)`.
GPSSignalQuality updateSignalQuality Качество GPS-сигнала.
ping ok Ответ на `pingEvent(token)`.

Команды `runEnum`

Команды передаются строкой: `window.androidApi.runEnum(TOKEN, "MEDIA_PLAY")`.

Группа Имена Описание
Навигация интерфейса GO_TO_PP, GO_TO_GU, TOGGLE_GU_PP, TOGGLE_GU_PP_CPP, GO_CPP_TO_PP, TOGGLE_CPP_PP Переключение между головным устройством, приборкой, картами и штатными экранами.
Системные действия RUN_BLACK, OPEN_SHTORKA, CLOSE_SHTORKA, GLOBAL_BACK, GLOBAL_HOME, RUN_START_APP_MENU, RUN_FUN_CAR, SPLIT_RUN Чёрный экран, шторка, глобальные кнопки, меню приложений, быстрый доступ к функциям авто и разделение экрана.
Медиа MEDIA_PLAY, MEDIA_PAUSE, MEDIA_NEXT, MEDIA_BLACK Управление воспроизведением: плей, пауза, следующий и предыдущий трек.
Передние сиденья heat_seat_l_0..3, heat_seat_r_0..3, vent_seat_l_0..3, vent_seat_r_0..3 Подогрев и вентиляция водительского и пассажирского передних сидений.
Обогревы heat_wheel_on, heat_wheel_off, heat_windshield_on, heat_windshield_off, heat_rearwindow_on, heat_rearwindow_off Руль, лобовое стекло и заднее стекло.
Задние сиденья heat_zad_seat_l_0..3, heat_zad_seat_r_off, heat_zad_seat_r_1..3 Подогрев заднего левого и заднего правого сиденья.
Профили и прочее VIBOR_VODITEL, voditel_seat_1, voditel_seat_2, voditel_seat_3, VIEW_ALL_MESSAGE, b_fiksik_on, b_fiksik_off, RUN_APP_MORE, RUN_SPICH_FOCUS, Recirculation_On, Recirculation_Off Выбор водителя, сообщения, голосовой помощник, группа задач, активация голосового помощника и рециркуляция.

Примеры

Минимальные вызовы для HTML-экрана.

Получить текущую громкость

const volume = window.androidApi.getvol(TOKEN);
if (volume >= 0) {
  console.log("Media volume:", volume + "%");
}

Получить передние сиденья

const seats = JSON.parse(
  window.androidApi.getCarData(TOKEN, "frontSeats")
);

console.log(seats.frontLeft.heat);
console.log(seats.frontRight.vent);

Открыть приложение из слота

const app = JSON.parse(
  localStorage.getItem("app_slot_1")
);

if (app) {
  window.androidApi.runApp(TOKEN, app.package);
}

Принять события Android

window.onAndroidEvent = function(type, data) {
  if (typeof data === "string") {
    data = JSON.parse(data);
  }

  if (type === "musicInfo") {
    title.textContent = data.SongName || "";
  }
};

Получать строки из логов авто

window.androidApi.setLogFilter(
  TOKEN,
  "VehicleSpeed",
  "SeatHeat",
  "HVAC"
);

window.onAndroidEvent = function(type, data) {
  if (typeof data === "string") {
    data = JSON.parse(data);
  }

  if (type === "LogData") {
    console.log(data.value);
  }
};
